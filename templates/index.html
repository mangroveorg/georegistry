Introduction:


Welcome to the Georegistry, the geospatial health repository developed by the Earth Institute at Columbia University. What makes Georegistry so special is that it is deisgned specifically to result high-quality, vetted, health information from multiple sources (mobile devices, spreadsheets, web sites, etc.). In short, Georegistry makes it easy for govenments and NGOs to share, visualize, and report information.

Georegistry provides a standardized way to collect and store health data that is too often kept in spreadsheets in incompatiable formats. To see a client example visit http://www.georegistry.org. You can also have a look at the developer documentation, or request access to the application programming interface(API).

Currently, we are only accepting input from governments and NGOs sources for the following countries:

  • Nigeria
  • Ghana



Georegistry Capabilities:


  • Store and search multiple spellings or langauages for a single item
  • Duplicate detection
  • Flag items as "offical" or government verified
  • Historical storage and search
  • Spreadsheet import tools
  • Developer Friendly: RESTful API serving GeoJSON, open source, Client Examples with Source Code
  • Connect via mobile decvices, web, xforms, or other application
